# Agent Guidelines — Confetti Docs

## Overview

This is a **VitePress** documentation site for the Confetti API. It uses VitePress v1.x with the default theme, customized with Confetti brand colors and typography.

## Key Principles

### Prefer VitePress built-in configuration

Always use VitePress's own theming system before resorting to CSS overrides:

- **Colors and branding** — Use `--vp-c-brand-*` CSS custom properties in `:root` / `.dark` selectors. VitePress provides a full set of design tokens; see https://vitepress.dev/reference/default-theme-config.
- **Layout and navigation** — Configure via `.vitepress/config.mts` (`themeConfig.nav`, `themeConfig.sidebar`, etc.) rather than hiding/rearranging elements with CSS.
- **Typography** — Set `--vp-font-family-base` and related variables instead of overriding individual selectors.
- **Components** — Extend the theme through `.vitepress/theme/index.ts` (registering Vue components, using slots) instead of patching markup with CSS.

### Minimize overrides in `custom.css`

The file `.vitepress/theme/custom.css` already contains a fair amount of `!important` overrides for sidebar spacing, hero layout, and font sizes. **Avoid adding more.** When making visual changes:

1. First check if VitePress exposes a CSS variable or config option for it.
2. If a variable exists, set it in the `:root` / `.dark` blocks at the top of `custom.css`.
3. Only write selector-based overrides as a last resort, and document why.

### Docs are auto-generated

The Markdown files under `api/` (except `api/index.md`) are generated by `scripts/generate-docs.ts` from the `confetti` npm package metadata. **Do not hand-edit generated docs** — modify the generator script instead.

Run `npm run docs:generate` to regenerate, then `npm run docs:dev` to preview.

### Project structure

```
.vitepress/
  config.mts          — VitePress site config (nav, sidebar, plugins)
  theme/
    index.ts           — Theme entry; extends default theme, registers components
    custom.css         — Brand colors, fonts, and layout tweaks
    components/        — Custom Vue components (e.g. ApiEndpoint)
api/                   — Generated API reference pages (do not edit by hand)
scripts/
  generate-docs.ts     — Reads confetti package and outputs Markdown
index.md               — Home page
```

### Tech stack

- **VitePress** v1.x (static site generator built on Vite + Vue 3)
- **Vue 3** for custom components
- **TypeScript** for scripts and config
- **Node.js** ≥ 24
- `vitepress-plugin-llmstxt` for LLM-friendly output
- `lucide-vue-next` for sidebar icons
